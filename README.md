### About

I came to engineering from an unusual direction: I trained as a **historian**, spent ~5 years in **US-facing logistics operations**, and taught myself the data stack along the way. That path turns out to be an advantage — five years reading freight tariffs, TMS exports and receivables data means I rarely have to ask what a column *means* before I model it.

Today I work on **data engineering and applied AI**: batch and streaming pipelines on GCP, retrieval-augmented generation over document corpora, and multi-agent systems. I'm currently finishing an **MSc in Data Science** (Universidad Simón Bolívar) and preparing for the **Google Cloud Professional Data Engineer** certification.

- 🔭 Building RAG pipelines and agent workflows that run in production, not in notebooks
- 🌱 Currently going deeper into **MLOps** and **AI systems design**
- 🎯 Next step: a master's in **Artificial Intelligence** — formalizing what I've been building by hand
- 💬 Ask me about **RAG, vector databases, GCP data pipelines** — or about medieval historiography
